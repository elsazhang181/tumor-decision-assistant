## 小程序 + H5 Webview 部署流程

### Step 1: 部署 H5 应用

H5 应用已通过 Coze 平台部署，当前可访问上述地址。

### Step 2: 配置微信小程序

1. 打开 `miniprogram/project.config.json`，修改 `appid` 为您的小程序 AppID

2. 打开 `miniprogram/pages/index/index.js`，确认 `targetUrl` 已设置为您的 H5 地址：
   ```javascript
   targetUrl = 'https://您的H5地址';
   ```

### Step 3: 域名白名单配置

在微信公众平台配置业务域名：
1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 进入「开发」→「开发管理」→「开发设置」
3. 在「业务域名」中添加您的 H5 域名
4. 下载校验文件并放置到 H5 项目的 `public` 目录

### Step 4: 上传校验文件

如果需要业务域名校验：
1. 从微信公众平台下载校验文件（如 `WW_verify_xxxx.txt`）
2. 将文件放入 H5 项目的 `public` 目录
3. 确保可通过 `https://您的域名/WW_verify_xxxx.txt` 访问

### Step 5: 编译小程序

```bash
cd miniprogram
# 使用微信开发者工具打开项目
# 点击「编译」进行预览
# 点击「上传」发布审核
```

## 扣子 Bot 配置

### 1. 知识库导入

1. 登录扣子平台
2. 导入 `知识库/扣子导入_肠癌肝转移_QA对.csv` 作为知识库
3. 配置字段映射：
   - 问题字段 → 标准问题
   - 答案字段 → 答案
   - 召回阈值：0.75

### 2. 引用来源设置（建立患者信任）

**必须开启以下设置：**

1. **开启引用来源显示**
   - 在Bot的「知识库」设置中
   - ✅ 开启「显示引用来源」

2. **设置来源标签**（参考 `知识库/引用来源设置.json`）

   | 来源类型 | 标签格式 | 示例 |
   |---------|---------|------|
   | CSCO指南 | 【CSCO指南2025】 | 【CSCO指南2025】... |
   | NCCN指南 | 【NCCN指南2026】 | 【NCCN指南2026】... |
   | 医院官方 | 【来源：XX医院官网】 | 【来源：北京大学肿瘤医院官网】 |
   | 政府机构 | 【来源：国家卫健委】 | 【来源：国家卫健委】... |
   | 知识库-专家 | 【知识库：熊猫群】 | 【知识库：熊猫群】... |
   | 知识库-保险 | 【知识库：保险】 | 【知识库：保险】... |

3. **设置免责声明**（回答末尾必须包含）

   ```
   **📋 信息来源声明**
   本回答参考了以下来源：
   • 【CSCO指南2025】2025 CSCO结直肠癌诊疗指南
   • 【知识库：熊猫群】肿瘤专家信息汇总
   • 【来源：国家卫健委】国家卫生健康委员会官网

   **⚠️ 重要提示**
   以上信息仅供参考，不能替代专业医生的诊断和治疗。
   如有不适，请尽快就医；具体治疗方案请遵医嘱。
   ```

详细配置请参考 `知识库/引用来源配置指南.md`

| 模块 | 数据来源 | 说明 |

### 回答流程（新增）

1. **综合知识库检索**
   - Step 1: CSCO指南检索（诊断标准、分期、化疗方案等）
   - Step 2: 专家知识库检索（手术、转化治疗等）
   - Step 3: 保险知识库检索（涉及费用/保险时）
   - Step 4: 官方信息来源引用

2. **官方信息来源**
   - 医院官方：官网公示、就诊指南
   - 医疗行业：中华医学会、CSCO指南、NCCN指南
   - 政府机构：国家卫健委、国家医保局

3. **医患沟通提问清单**
   - 根据问题类型生成针对性提问清单
   - 诊断相关：问分期、问依据
   - 治疗相关：问方案、问效果、问副作用
   - 检查相关：问必要性、问费用、问时间

4. **免责声明**
   - 不涉及医疗诊断
   - 不提供诊疗方案
   - 仅作为决策辅助工具
   - 具体决策请遵医嘱

## 上下文保留机制

### 功能说明
在四个环节切换时，自动保留和传递上下文信息，确保检测连贯性。

### 实现原理
1. **上下文提取**：每个环节对话结束后，提取关键信息（症状分析、诊断结论、推荐科室等）
2. **上下文存储**：使用 React state 管理对话上下文
3. **上下文传递**：切换环节时，将前序环节的关键结论传递给新环节
4. **上下文注入**：在 API 请求的 system prompt 中注入前序上下文

### 上下文数据结构
```typescript
interface ConversationContext {
  symptoms?: string;           // 症状描述
  symptomAnalysis?: string;    // 症状分析结论
  urgencyLevel?: string;      // 紧急程度评估
  suspectedCondition?: string; // 疑似病症
  recommendedDepartments?: string[]; // 推荐科室
  recommendedHospitals?: string[];   // 推荐医院
  treatmentStage?: string;     // 治疗阶段
  summary?: string;           // 对话摘要
}
```

### 用户体验
- 症状自查环节输入症状 → 切换到科室匹配环节
- 系统自动提示："您刚才描述的症状是...，建议就诊科室是..."
- 无需重复描述症状，直接获取科室推荐
